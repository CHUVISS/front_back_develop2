## 📝 О проекте

ShopApp — это полнофункциональное веб-приложение для электронной коммерции с системой аутентификации, авторизации и управления доступом на основе ролей (RBAC). Проект разработан в рамках практического занятия №11 по дисциплине "Фронтенд и бэкенд разработка".

### Ключевые особенности:
- ✅ JWT аутентификация с access/refresh токенами
- ✅ Ролевая модель доступа (Admin, Seller, User)
- ✅ Управление товарами (CRUD)
- ✅ Управление пользователями (только Admin)
- ✅ Светлая и тёмная темы
- ✅ Адаптивный дизайн
- ✅ Валидация форм
- ✅ Защита маршрутов

## 🔐 Роли и права доступа

| Маршрут | Метод | Гость | User | Seller | Admin |
|---------|-------|-------|------|--------|-------|
| `/api/auth/register` | POST | ✅ | ✅ | ✅ | ✅ |
| `/api/auth/login` | POST | ✅ | ✅ | ✅ | ✅ |
| `/api/auth/refresh` | POST | ✅ | ✅ | ✅ | ✅ |
| `/api/auth/me` | GET | ❌ | ✅ | ✅ | ✅ |
| `/api/users` | GET | ❌ | ❌ | ❌ | ✅ |
| `/api/users/:id` | GET/PUT/DELETE | ❌ | ❌ | ❌ | ✅ |
| `/api/products` | GET | ❌ | ✅ | ✅ | ✅ |
| `/api/products` | POST | ❌ | ❌ | ✅ | ✅ |
| `/api/products/:id` | GET | ❌ | ✅ | ✅ | ✅ |
| `/api/products/:id` | PUT | ❌ | ❌ | ✅ | ✅ |
| `/api/products/:id` | DELETE | ❌ | ❌ | ❌ | ✅ |

**Обозначения:**
- ✅ — доступ разрешён
- ❌ — доступ запрещён

### Описание ролей:
- **Гость** — не аутентифицированный пользователь
- **User (Пользователь)** — пользователь сайта (только просмотр товаров)
- **Seller (Продавец)** — сотрудник сайта (добавление и редактирование товаров)
- **Admin (Администратор)** — управленец сайта (права продавца + управление пользователями)

## 📦 Установка

### Требования
- Node.js 18.x или выше
- npm 9.x или выше

### 1. Клонирование репозитория
```bash
git clone <URL-репозитория>
cd <имя-папки-проекта>
```

### 2. Установка зависимостей бекенда
```bash
cd server
npm install
```

### 3. Установка зависимостей фронтенда
```bash
cd ../client
npm install
```

### 4. Настройка переменных окружения (опционально)
Создайте файл `.env` в папке `server`:
```env
PORT=3000
ACCESS_SECRET=access_secret
REFRESH_SECRET=refresh_secret
ACCESS_EXPIRES_IN=15m
REFRESH_EXPIRES_IN=7d
```

## 🚀 Запуск

### Запуск бекенда
```bash
cd server
npm start
```
Сервер запустится на `http://localhost:3000`

### Запуск фронтенда (в новом терминале)
```bash
cd client
npm run dev
```
Приложение откроется на `http://localhost:3001` (или другой порт, указанный в терминале)

### Документация API
Откройте в браузере: `http://localhost:3000/api-docs`

## 🌐 API Endpoints

### Аутентификация
| Метод | Endpoint | Описание | Доступ |
|-------|----------|----------|--------|
| POST | `/api/auth/register` | Регистрация пользователя | Гость |
| POST | `/api/auth/login` | Вход в систему | Гость |
| POST | `/api/auth/refresh` | Обновление токенов | Гость |
| GET | `/api/auth/me` | Получение данных пользователя | User/Seller/Admin |

### Пользователи (Admin only)
| Метод | Endpoint | Описание | Доступ |
|-------|----------|----------|--------|
| GET | `/api/users` | Список всех пользователей | Admin |
| GET | `/api/users/:id` | Данные пользователя по ID | Admin |
| PUT | `/api/users/:id` | Обновление пользователя | Admin |
| DELETE | `/api/users/:id` | Блокировка пользователя | Admin |

### Товары
| Метод | Endpoint | Описание | Доступ |
|-------|----------|----------|--------|
| GET | `/api/products` | Список товаров | User/Seller/Admin |
| GET | `/api/products/:id` | Товар по ID | User/Seller/Admin |
| POST | `/api/products` | Создать товар | Seller/Admin |
| PUT | `/api/products/:id` | Обновить товар | Seller/Admin |
| DELETE | `/api/products/:id` | Удалить товар | Admin |

## 🔑 Учётные данные по умолчанию

После первого запуска сервера автоматически создаётся учётная запись администратора:

| Роль | Email | Пароль |
|------|-------|--------|
| Admin | `admin@shopapp.com` | `admin123` |

**Важно:** Пароль хэшируется при первом запуске. Не меняйте хэш вручную в коде.

## 🎨 Темы оформления

Приложение поддерживает две темы оформления:

- **Светлая тема** (по умолчанию) — классический светлый дизайн
- **Тёмная тема** — комфортный режим для работы в условиях низкой освещённости

Переключение тем осуществляется через кнопку 🌙/☀️ в навигационной панели. Выбор темы сохраняется в `localStorage` и восстанавливается при следующем посещении.

## ⚠️ Примечания

1. Проект предназначен для учебных целей
2. В production-среде необходимо:
   - Заменить секреты JWT на надёжные
   - Настроить HTTPS
   - Добавить rate limiting
   - Реализовать валидацию данных на сервере
   - Добавить обработку ошибок
3. Данные хранятся в памяти сервера и сбрасываются при перезапуске
4. Для постоянного хранения данных рекомендуется подключить базу данных (MongoDB, PostgreSQL)
